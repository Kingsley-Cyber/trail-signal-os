# Expansion Roadmap

## Phase 1 — Manual evidence excellence

- Stabilize ontology and IDs.
- Complete ten research runs across distinct activity domains.
- Calibrate scoring against rejected and advanced candidates.
- Build a reusable complaint/workaround lexicon.

## Phase 2 — Source adapters

- Add credentialed adapters for official APIs where access is available.
- Add compliant importers for platform exports and analyst CSVs.
- Build deduplication by canonical URL, listing ID and independence group.
- Add freshness alerts and stale-candidate transitions.

## Phase 3 — Semantic system

- Embed activity, task, friction and evidence objects separately.
- Build alias and predicate registries.
- Add cross-domain motif discovery: wet-hand operation, occupied-hand access, dirty/clean transition, cold dexterity, dark setup and moving-to-waiting transitions.
- Use LLMs for proposal and synthesis only; keep metrics deterministic.

## Phase 4 — Experiment learning loop

- Store creative, click, interview, waitlist, preorder and return outcomes.
- Compare predicted score with observed experiment performance.
- Recalibrate weights without deleting historical versions.
- Create domain-specific scorecards once enough outcomes exist.

## Phase 5 — Portfolio intelligence

- Track shared components and customer overlap.
- Detect seasonal portfolio balancing.
- Model bundle and expansion paths.
- Maintain a product thesis graph linking motifs across hobbies.
